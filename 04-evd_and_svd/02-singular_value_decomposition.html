Singular Value Decomposition

Definition

In the spirit of the eigenvalue decomposition for square matrices, we define the following decomposition for an arbitrary matrix A∈Cm×n: A=UΣV† where U∈Cm×m and V∈Cn×n are unitary matrices, and Σ∈Rm×n is a diagonal matrix with non-negative diagonal entries σ1​≥σ2​≥…≥σp​≥0, where p=min(m,n). This decomposition is known as the singular value decomposition (SVD).

But what does it mean for a rectangular matrix Σ to be diagonal? Strictly speaking, this is not possible, but we can bring the matrix into a form that resembles diagonal form, namely Σ=(D0​00​​). Thus, Σ contains a p×p diagonal matrix D and possibly zero matrices to fulfill the dimensions.

The diagonal entries σi​ are called singular values, and the columns of U and V are referred to as left and right singular vectors, respectively. In analogy to Equation {{eq: eq:eigenvalue_decomposition_normal}}, the singular value decomposition can be written as A​=UΣV†=​∣u1​∣​∣u2​∣​⋯​∣um​∣​​​σ1​0⋮00⋮0​0σ2​⋮00⋮0​⋯⋯⋱⋯⋯⋱⋯​000σp​0⋮0​00⋮00⋮0​⋯⋯⋱⋯⋯⋱⋯​00⋮00⋮0​​​​— v1†​ —— v2†​ —⋮— vn†​ —​​=σ1​u1​v1†​+σ2​u2​v2†​+⋯+σp​up​vp†​=i=1∑p​σi​ui​vi†​.​(3.6)​

One can see a clear difference to the eigenvalue decomposition: While in the eigenvalue decomposition all eigenvectors are important, here (m−p) or (n−p) singular vectors do not contribute to the decomposition. This means that one of the matrices U or V can be reduced without changing the decomposition. Assuming without loss of generality that m≥n, we can define U^∈Cm×n and Σ^∈Rn×n by removing the last (m−n) columns from U and the last (m−n) rows from Σ. We also define V^=V. Then the decomposition A=U^Σ^V^†, which is known as the economic singular value decomposition, holds. If one of the dimensions n,m is much larger than the other, the economic SVD is advantageous because it requires less memory.

Properties

We now consider the matrix products A†A and AA† using the SVD. We have A†AAA†​=(UΣV†)†UΣV†=(V†)†Σ†U†UΣV†=VΣ†ΣV†=UΣV†(UΣV†)†=UΣV†(V†)†Σ†U†=UΣΣ†U†​.(3.7)

Because the non-zero elements in Σ are only on the diagonal, the products Σ†Σ and ΣΣ† are diagonal matrices with the diagonal elements σi2​.

The two equations (3.7) are similar to the eigenvalue decomposition for normal matrices B=QΛQ−1. In fact, the products A†A and AA† are even hermitian, because (A†A)†(AA†)†​=A†(A†)†=A†A=(A†)†A†=AA†​​, which also makes them normal.

For this reason, the singular values correspond to the square roots of the eigenvalues of A†A and AA†. The left singular vectors are therefore the eigenvectors of AA†, while the right singular vectors represent the eigenvectors of A†A. We have thus established a connection between the SVD and the EVD.

Although Equation (3.7) is mathematically correct, one should not compute the SVD of A using the EVD of A†A and AA†:

Phase ambiguity

The eigenvectors of a matrix are only unique up to a (complex) factor. Even if one normalises the eigenvectors, the phase remains undetermined. For example, if one applies a minus sign to the first left singular vector u1​, the equations (3.7) remain valid. However, the SVD in Equation (3.6) is no longer correct, as the first term σ1​u1​v1†​ receives a minus sign. To keep the decomposition valid, the sign of v1​ would need to be adjusted. However, since the two EVDs are independent of each other, such adjustments are not possible. For this reason, one should not compute the SVD from the double EVD, unless only the singular values and not the singular vectors are needed. For the following considerations, it is sufficient to know that there are special algorithms that compute the left and right singular vectors simultaneously, which avoids this problem.

Use with NumPy

The singular value decomposition is available in NumPy via the function numpy.linalg.svd. As an argument, one passes the matrix A for which the SVD should be computed. The return values are the matrices U, Σ, and V†. With the optional argument full_matrices=False, the economic SVD is computed. The default option is full_matrices=True. An example call looks like this:

import numpy as np

a = np.random.rand(9, 6)
u, s, vh = np.linalg.svd(a, full_matrices=False)
print(u.shape) # (9, 6)
print(s.shape) # (6,)
print(vh.shape) # (6, 6)
